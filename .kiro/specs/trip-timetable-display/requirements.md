# 要件定義書

## はじめに

本機能は、リアルタイム車両追跡機能で表示される便ID（trip_id）から、その便の全停車バス停と到着時刻を時刻表形式で表示する機能です。ユーザーは車両マーカーをクリックすることで、その便がどのバス停に何時に到着するかを一目で確認でき、乗車計画を立てやすくなります。

## 用語集

- **System**: 佐賀バスナビゲーションアプリケーション
- **trip_id**: GTFS標準の便識別子。特定の運行便を一意に識別するキー
- **stop_times.txt**: GTFSデータの一部。各便の停車バス停と到着・出発時刻を記録
- **stops.txt**: GTFSデータの一部。バス停の名称と位置情報を記録
- **車両マーカー**: 地図上に表示されるバスの現在位置を示すマーカー
- **吹き出し**: 車両マーカーをクリックした際に表示される情報ウィンドウ
- **始点**: 便の最初の停車バス停
- **終点**: 便の最後の停車バス停
- **到着時刻**: バスが各バス停に到着する予定時刻（HH:MM形式）

## 要件

### 要件1: 便の時刻表データ取得

**ユーザーストーリー:** バス利用者として、選択した便の全停車バス停と到着時刻を知りたい。そうすることで、どのバス停で乗降できるかを把握できる。

#### 受入基準

1. WHEN 車両マーカーがクリックされた際、THE System SHALL trip_idを使用してstop_times.txtから該当便の全停車時刻データを取得する
2. WHEN 停車時刻データを取得する際、THE System SHALL stop_sequenceの昇順でソートする
3. WHEN 停車時刻データを取得する際、THE System SHALL 各stop_idに対応するバス停名をstops.txtから取得する
4. WHEN バス停名を取得する際、THE System SHALL stop_idが存在しない場合は「バス停名不明」と表示する
5. WHEN 到着時刻を表示する際、THE System SHALL arrival_timeフィールドからHH:MM形式で時刻を抽出する

### 要件2: 時刻表の表示形式

**ユーザーストーリー:** バス利用者として、時刻表を見やすい形式で確認したい。そうすることで、情報を素早く理解できる。

#### 受入基準

1. WHEN 時刻表を表示する際、THE System SHALL 「始点のバス停（到着HH:MM）→ 次のバス停（到着HH:MM）→ ... → 終点のバス停（到着HH:MM）」の形式で表示する
2. WHEN 時刻表を表示する際、THE System SHALL 各バス停情報を矢印（→）で区切る
3. WHEN 時刻表を表示する際、THE System SHALL バス停名と到着時刻を括弧で囲んで表示する
4. WHEN 時刻表が長い場合、THE System SHALL 適切な改行を挿入して可読性を保つ
5. WHEN 時刻表を表示する際、THE System SHALL 便IDと路線名を時刻表の上部に表示する

### 要件3: 吹き出しへの統合

**ユーザーストーリー:** バス利用者として、車両の運行状態と時刻表を同じ場所で確認したい。そうすることで、情報を探す手間が省ける。

#### 受入基準

1. WHEN 車両マーカーの吹き出しを表示する際、THE System SHALL 既存の運行状態情報の下に時刻表を追加する
2. WHEN 時刻表を表示する際、THE System SHALL 「時刻表」というラベルを付けて区別する
3. WHEN 吹き出しを表示する際、THE System SHALL 時刻表が長い場合でも全体が表示されるようにスクロール可能にする
4. WHEN 吹き出しを表示する際、THE System SHALL 時刻表部分を視覚的に区別するためのスタイルを適用する
5. WHERE 時刻表データが取得できない場合、THE System SHALL 「時刻表情報が取得できません」と表示する

### 要件4: パフォーマンス最適化

**ユーザーストーリー:** バス利用者として、時刻表を素早く表示してほしい。そうすることで、待ち時間なく情報を確認できる。

#### 受入基準

1. WHEN 時刻表データを取得する際、THE System SHALL DataLoaderのキャッシュされたstop_timesとstopsデータを使用する
2. WHEN 時刻表を生成する際、THE System SHALL 不要なDOM操作を避けて文字列結合で生成する
3. WHEN 同じ便の時刻表を再度表示する際、THE System SHALL 生成済みの時刻表をキャッシュから取得する
4. WHEN 時刻表データの処理に100ms以上かかる場合、THE System SHALL 「読み込み中...」と表示する
5. WHEN 時刻表の生成が完了した際、THE System SHALL 処理時間をコンソールにログ出力する

### 要件5: エラーハンドリング

**ユーザーストーリー:** バス利用者として、エラーが発生しても基本機能を使い続けたい。そうすることで、アプリケーションの可用性が保たれる。

#### 受入基準

1. WHEN trip_idに対応するstop_timesデータが存在しない場合、THE System SHALL 「時刻表情報が見つかりません」と表示する
2. WHEN stop_timesデータの取得中にエラーが発生した場合、THE System SHALL エラーをコンソールに記録し、ユーザーには「時刻表情報の取得に失敗しました」と表示する
3. WHEN 時刻表の生成中にエラーが発生した場合、THE System SHALL 既存の運行状態情報は引き続き表示する
4. WHEN DataLoaderが初期化されていない場合、THE System SHALL 「データ読み込み中です」と表示する
5. WHEN エラーが発生した場合、THE System SHALL エラーの詳細情報（trip_id、エラーメッセージ）をコンソールに出力する

### 要件6: 現在位置の強調表示

**ユーザーストーリー:** バス利用者として、時刻表の中で現在バスがどこにいるかを知りたい。そうすることで、残りの停車バス停を把握できる。

#### 受入基準

1. WHERE 車両位置情報にcurrent_stop_sequenceが含まれる場合、THE System SHALL 該当するバス停を時刻表内で強調表示する
2. WHEN 現在位置のバス停を強調表示する際、THE System SHALL 太字または色を変えて視覚的に区別する
3. WHEN 現在位置のバス停を強調表示する際、THE System SHALL 「← 現在地」などのマーカーを追加する
4. WHERE 車両が運行開始前の場合、THE System SHALL 始点のバス停を強調表示する
5. WHERE 車両が運行終了後の場合、THE System SHALL 終点のバス停を強調表示する

### 要件7: 時刻表の折りたたみ機能

**ユーザーストーリー:** バス利用者として、時刻表が長い場合は折りたたんで表示したい。そうすることで、吹き出しがコンパクトになり地図が見やすくなる。

#### 受入基準

1. WHEN 時刻表の停車バス停数が10個を超える場合、THE System SHALL デフォルトで折りたたんだ状態で表示する
2. WHEN 時刻表が折りたたまれている場合、THE System SHALL 「時刻表を表示（全○停車）」というリンクを表示する
3. WHEN 折りたたみリンクがクリックされた場合、THE System SHALL 時刻表を展開して全停車バス停を表示する
4. WHEN 時刻表が展開されている場合、THE System SHALL 「時刻表を折りたたむ」というリンクを表示する
5. WHEN 折りたたみリンクがクリックされた場合、THE System SHALL 時刻表を折りたたんで最初の3停車と最後の3停車のみを表示する
