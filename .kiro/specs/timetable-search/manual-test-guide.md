## 1. 正常系テスト

### 1.1 佐賀駅バスセンター → 佐賀大学（平日、現在時刻）

**目的**: 最も一般的な検索パターンの動作確認

**前提条件**:
- 現在時刻が平日の6:00〜21:00の間であること
- データが正常に読み込まれていること

**テスト手順**:
1. 乗車バス停フィールドに「佐賀駅」と入力
2. サジェストリストから「佐賀駅バスセンター」を選択
3. 降車バス停フィールドに「佐賀大学」と入力
4. サジェストリストから「佐賀大学」を選択
5. 時刻オプションで「今すぐ」を選択
6. 「検索」ボタンをクリック

**期待される結果**:
- [ ] サジェストリストが正しく表示される（1文字入力後）
- [ ] 検索ボタンが有効になる（両方のバス停選択後）
- [ ] 検索結果が2秒以内に表示される
- [ ] 検索結果に以下の情報が含まれる:
  - [ ] 出発時刻（現在時刻以降）
  - [ ] 到着時刻
  - [ ] 所要時間（分）
  - [ ] 運賃（大人/小人）
  - [ ] 事業者名
  - [ ] 路線名
- [ ] 検索結果が出発時刻の昇順で表示される
- [ ] 最大20件まで表示される

**判定**: ☐ 合格 / ☐ 不合格

## 付録: テスト実施のヒント

### データ読み込み確認方法

ブラウザの開発者ツール（F12）を開き、Consoleタブで以下を確認:
```javascript
// データが読み込まれているか確認
console.log('Bus Stops:', busStops.length);
console.log('Timetable:', timetable.length);
console.log('Fares:', fares.length);
```

### パフォーマンス計測方法

ブラウザの開発者ツール（F12）を開き、Performanceタブで:
1. 記録開始ボタンをクリック
2. 操作を実行（検索など）
3. 記録停止ボタンをクリック
4. タイムラインで処理時間を確認

### レスポンシブテスト方法（Chrome DevTools）

2. デバイスツールバーを切り替え（Ctrl+Shift+M）

### NTP接続確認方法

ブラウザの開発者ツール（F12）を開き、Networkタブで:
1. 「今すぐ」検索を実行
2. `ntp-a1.nict.go.jp` へのリクエストを確認
3. レスポンスステータスが200であることを確認
4. レスポンスボディにタイムスタンプが含まれることを確認

### 祝日判定確認方法

ブラウザの開発者ツール（F12）を開き、Consoleタブで:
```javascript
// 祝日カレンダーが読み込まれているか確認
console.log('Holiday Cache:', holidayCache);

// 特定の日付が祝日かチェック
const testDate = new Date('2025-01-01');
isHoliday(testDate).then(result => console.log('Is Holiday:', result));
```
