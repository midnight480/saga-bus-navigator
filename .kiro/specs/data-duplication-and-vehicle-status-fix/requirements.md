# 要件定義書

## はじめに

佐賀バスナビゲーターアプリにおいて、時刻表データが重複表示される問題と、運行終了したバスのリアルタイム位置情報が表示され続ける問題が報告されています。これらの問題を解決し、ユーザーに正確で有用な情報を提供します。

## 用語集

- **GTFS**: General Transit Feed Specification - 公共交通機関の時刻表データの標準形式
- **DataLoader**: GTFSデータを読み込み、アプリケーション形式に変換するモジュール
- **RealtimeVehicleController**: リアルタイム車両位置情報を管理・表示するコントローラー
- **時刻表データ**: バス停、路線、時刻などの静的な運行情報
- **運行終了状態**: バスが最終停留所に到着し、運行を完了した状態
- **車両マーカー**: 地図上に表示されるバスの位置を示すアイコン

## 要件

### 要件1: 時刻表データ重複表示の修正

**ユーザーストーリー**: ユーザーとして、時刻表を検索したとき、同じ行き先・時刻のバスが2つずつ表示されず、正確に1つだけ表示されることを期待します。

#### 受入基準

1. WHEN GTFSデータを読み込むとき THEN システムは各ZIPファイルを1回のみ処理すること
2. WHEN 時刻表データを変換するとき THEN システムは各stop_timesレコードを1回のみ処理すること
3. WHEN 時刻表データをキャッシュするとき THEN システムは重複したエントリを含まないこと
4. WHEN ユーザーが時刻表を検索するとき THEN システムは各便を1回のみ表示すること
5. WHEN データローダーが複数回呼び出されるとき THEN システムはキャッシュされたデータを再利用し、重複して追加しないこと

### 要件2: 運行終了バスの非表示

**ユーザーストーリー**: ユーザーとして、リアルタイム位置情報を確認するとき、運行終了したバスは表示されず、現在運行中のバスのみが表示されることを期待します。

#### 受入基準

1. WHEN バスの運行状態が「運行終了」と判定されるとき THEN システムはそのバスの車両マーカーを地図上に表示しないこと
2. WHEN 運行中のバスが運行終了状態に遷移するとき THEN システムは既存の車両マーカーを地図から削除すること
3. WHEN リアルタイムデータを更新するとき THEN システムは運行終了状態のバスをフィルタリングすること
4. WHEN ユーザーが地図を表示するとき THEN システムは運行中のバスのみを表示すること
5. WHEN 運行終了の判定を行うとき THEN システムは最終停留所の到着予定時刻と現在時刻を比較すること

### 要件3: データ処理の検証とログ出力

**ユーザーストーリー**: 開発者として、データ処理の各段階で正確性を検証し、問題を早期に発見できることを期待します。

#### 受入基準

1. WHEN GTFSデータを読み込むとき THEN システムは読み込んだレコード数をログ出力すること
2. WHEN データ変換を実行するとき THEN システムは変換前後のレコード数をログ出力すること
3. WHEN 重複データを検出したとき THEN システムは警告ログを出力すること
4. WHEN 車両マーカーを更新するとき THEN システムは運行状態をログ出力すること
5. WHEN 運行終了バスをフィルタリングするとき THEN システムはフィルタリングされたバス数をログ出力すること
