# ORS APIキー管理ドキュメント

## APIキーの取得方法

### 1. OpenRouteServiceアカウントの作成

1. [OpenRouteService](https://openrouteservice.org/) のウェブサイトにアクセス
2. 「Sign Up」または「Get API Key」をクリック
3. アカウントを作成（無料プランで利用可能）

### 2. APIキーの取得

1. ログイン後、ダッシュボードにアクセス
2. 「API Keys」セクションでAPIキーを確認または生成
3. APIキーをコピー

### 3. APIキーの設定

#### ローカル開発環境

`.dev.vars`ファイルに設定:

```bash
ORS_API_KEY=your-api-key-here
```

#### 本番環境（Cloudflare Pages）

1. Cloudflare Pagesダッシュボードにアクセス
2. プロジェクトの「Settings」→「Environment variables」を開く
3. 「Add variable」をクリック
4. 変数名: `ORS_API_KEY`、値: あなたのAPIキーを入力
5. 「Encrypt」オプションを有効にする（推奨）
6. 「Save」をクリック

または、コマンドラインから:

```bash
./scripts/apply-cloudflare-secrets.sh
```

## レート制限の説明

### 無料プランの制限

- **分次制限**: 1分あたり最大40リクエスト
- **日次制限**: 1日あたり最大2000リクエスト

### 制限に達した場合の動作

1. **分次制限**: 最大4秒待機してから再試行（`maxWaitOnMinuteLimitMs`で設定）
2. **日次制限**: エラーメッセージを表示し、フォールバック（直線描画）を使用

### 制限の確認方法

- ORSダッシュボードでAPIキーの使用状況を確認
- ブラウザの開発者ツールのネットワークタブでリクエスト数を確認

## キャッシュ戦略の説明

### キャッシュの仕組み

1. **保存場所**: ブラウザの`localStorage`
2. **有効期限**: デフォルト24時間（`cacheTtlMs`で設定可能）
3. **キャッシュキー**: 座標配列から生成（同じ座標配列は同じキー）

### キャッシュの利点

- **API呼び出しの削減**: 同じ経路は再取得せず、キャッシュから取得
- **パフォーマンス向上**: キャッシュからの取得は即座に完了
- **レート制限の回避**: キャッシュヒット時はAPIを呼び出さないため、レート制限に達しにくい

### キャッシュのクリア

ユーザーがブラウザのデータをクリアすると、キャッシュも削除されます。

開発中にキャッシュをクリアする場合:

```javascript
// ブラウザのコンソールで実行
localStorage.clear(); // 全てのlocalStorageをクリア
// または
Object.keys(localStorage)
  .filter(key => key.startsWith('ors_route_'))
  .forEach(key => localStorage.removeItem(key)); // ORSキャッシュのみクリア
```

## セキュリティベストプラクティス

### APIキーの保護

1. **Gitにコミットしない**: `.dev.vars`は`.gitignore`に含まれています
2. **環境変数で管理**: 本番環境では環境変数を使用
3. **暗号化**: Cloudflare Pagesでは「Encrypt」オプションを有効にする

### レート制限の管理

1. **キャッシュの活用**: 可能な限りキャッシュを使用してAPI呼び出しを削減
2. **デバウンス**: 連続リクエストをデバウンスして、不要なAPI呼び出しを防止
3. **エラーハンドリング**: レート制限エラー時は適切にフォールバックを使用

## トラブルシューティング

### APIキーが認識されない

1. `.dev.vars`ファイルが正しい場所にあるか確認
2. `runtime-env.js`が正しく読み込まれているか確認（ブラウザの開発者ツールで確認）
3. 環境変数が正しく設定されているか確認

### レート制限エラーが頻繁に発生する

1. キャッシュが機能しているか確認
2. `debounceMs`を増やして、連続リクエストを減らす
3. `cacheTtlMs`を増やして、キャッシュの有効期限を延長

### キャッシュが機能しない

1. ブラウザの`localStorage`が有効か確認
2. キャッシュキーが正しく生成されているか確認（座標の順序が重要）
3. TTLが過ぎていないか確認
